# SQL QA Portfolio

## Overview

This project demonstrates practical SQL skills used in Software Quality Assurance (QA). It includes database validation, business rule verification, data integrity checks, and SQL queries commonly used during bug investigations.

The portfolio was created to simulate real-world QA database testing scenarios and demonstrate how SQL is used to validate application data.

---

## Project Database

The project contains the following tables:

- Users
- Products
- Orders

These tables are used to perform realistic QA validation scenarios.

## QA Test Cases Included

This portfolio contains **40 SQL QA Test Cases**, including:

- Duplicate email validation

### Product Validation
- Active vs inactive products
- Out-of-stock validation
- Negative price validation
- Duplicate product detection
- Product sorting

### Order Validation
- Positive quantity validation
- Existing user validation
- Existing product validation
- Customer order verification
- Users without orders

### Business Rule Validation
- Active products with zero stock
- Calculated order totals
- Total order value per user
- Orders per user
- Duplicate order detection
- Data integrity verification
